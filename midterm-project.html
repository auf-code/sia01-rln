1. Project Overview

Analyze a real-world systems problem and produce a formal Systems Architecture and Systems Integration document. Think like an architect: define boundaries, identify subsystems, model processes, and justify decisions.

2. Project Objectives

  • Analyze real-world problems from a systems perspective.
  • Design high-level system architectures.
  • Model business processes and system interactions.
  • Communicate decisions using standard technical diagrams.

5. Required Document Outline

Your PDF must follow this structure:

I. Introduction & Problem

Explain the system's purpose, stakeholders, and current pain points.

II. System Context Diagram

Visual representation of actors and external systems. (Diagram Required)

III. Business Process Model (BPMN)

Show the end-to-end flow and system touchpoints. (Diagram Required)

IV. System Architecture Design

Define the structure (Monolithic vs. Microservices) and rationale. (Component/C4 Diagram Required)

V. Integration Design

Explain APIs, messaging, and data flow. (Sequence Diagram Required)

VI. Non-Functional & Risks

Address scalability, performance, and technical trade-offs.
