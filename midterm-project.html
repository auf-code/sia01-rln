1. Introduction

Purpose: Explain what the system is and why this document exists.
This document presents the proposed systems architecture and integration design for an Online Enrollment and Payment System for a university. The purpose is to describe the system context, architectural structure, and key design decisions prior to implementation. It is intended for stakeholders, system analysts, and architects.

2. Problem Description & Objectives

Purpose: Clearly define the problem being solved and the goals of the system.
Many universities rely on disconnected systems for enrollment and billing. These require manual reconciliation, resulting in delays and data inconsistencies. The objective is to integrate these services into a unified platform that supports real-time data exchange and improves operational efficiency.

9. Risks & Trade-offs

Purpose: Demonstrate architectural judgment and awareness.
Risk: Dependency on external payment gateway downtime.
Mitigation: Implementing retry logic and clear error messaging.
Trade-off: Monolithic architecture simplifies initial development but limits independent scaling.
